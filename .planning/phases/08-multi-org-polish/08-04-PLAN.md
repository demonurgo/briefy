## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /organizations | User-supplied name + slug cross auth boundary; validated server-side |
| Modal close (click-outside) | onClick on overlay closes modal — stopPropagation on inner div prevents accidental close |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04-01 | Spoofing | Inertia useForm POST | mitigate | CSRF token automatically included by Inertia useForm; `X-XSRF-TOKEN` header sent on every POST |
| T-08-04-02 | Tampering | Slug auto-generation | accept | Client-side slugify is convenience only; server validates with `alpha_dash|unique:organizations,slug` regardless of client input |
| T-08-04-03 | Information Disclosure | orgErrors display | accept | Validation errors (name required, slug taken) are intentional user feedback; no sensitive data exposed |
| T-08-04-04 | Denial of Service | Modal open/close on fast clicks | accept | React state management handles rapid state toggles safely; no server calls on open/close |
</threat_model>
