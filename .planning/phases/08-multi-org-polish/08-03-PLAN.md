From node_modules/@inertiajs/core/types/types.d.ts:
`UseFormSubmitOptions = Omit<VisitOptions, 'data'>` — 'data' is not in submit options in v3
Fix: setData then post, OR use transform():
```typescript
// WRONG (current):
post(route('clients.update', client.id), { data: { ...data, social_handles: cleanedHandles } });
// CORRECT (v3 pattern):
setData('social_handles', cleanedHandles);
post(route('clients.update', client.id));
```
Caveat: setData is async-ish (React setState) — if synchronous guarantee needed, use `transform()`:
```typescript
// Also valid if setData timing is a concern:
transform(data => ({ ...data, social_handles: cleanedHandles }));
post(route('clients.update', client.id));
```
Choose whichever the current Clients/Edit.tsx submit handler accommodates without restructuring.
