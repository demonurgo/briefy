```php
<?php
// (c) 2026 Briefy contributors — AGPL-3.0

namespace App\Http\Controllers;

use App\Models\Organization;
use Illuminate\Http\RedirectResponse;
use Illuminate\Http\Request;

class OrganizationController extends Controller
{
    /**
     * Store a newly created organization.
     *
     * Creates the org, attaches the authenticated user as owner (D-05),
     * auto-switches to the new org (D-03), and redirects to dashboard.
     */
    public function store(Request $request): RedirectResponse
    {
        $validated = $request->validate([
            'name' => 'required|string|max:255',
            'slug' => 'required|string|max:100|alpha_dash|unique:organizations,slug',
        ]);

        $org = Organization::create($validated);

        // Attach creator as owner — mirrors InvitationController pivot attach pattern (D-05)
        $org->users()->attach($request->user(), [
            'role'      => 'owner',
            'joined_at' => now(),
        ]);

        // Auto-switch to new org — same field write as SettingsController@switchOrg (D-03)
        $request->user()->update(['current_organization_id' => $org->id]);

        return redirect()->route('dashboard')->with('success', 'Organização criada com sucesso.');
    }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| auth middleware | All routes inside `['auth', 'verified']` group require authenticated, verified users |
| Validation layer | Laravel Request->validate() is the gate for name + slug; no raw SQL, no direct input to model |
| Pivot attach | Organization::users()->attach() is Eloquent-managed — no raw pivot insert |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Spoofing | POST /organizations | mitigate | Route inside `['auth', 'verified']` middleware — unauthenticated requests redirected to /login (confirmed by test_store_requires_authentication) |
| T-08-02-02 | Tampering | OrganizationController@store | mitigate | `$request->validate()` with `max:255`, `max:100`, `alpha_dash` on slug, `unique:organizations,slug` — prevents injection and slug collisions |
| T-08-02-03 | Elevation of Privilege | organization_user pivot | mitigate | Creator always attached as `owner` (D-05) via code constant, not user-supplied value — role cannot be user-controlled |
| T-08-02-04 | Information Disclosure | 422 validation errors | accept | Slug uniqueness error reveals that slug is taken — acceptable, slugs are semi-public identifiers |
