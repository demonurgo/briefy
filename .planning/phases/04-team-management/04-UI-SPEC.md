| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding (`px-1`) |
| sm | 8px | Button padding (`px-2 py-1.5`), input gap (`gap-2`) |
| md | 16px | Section inner padding (`p-4`), form field spacing (`space-y-4`) |
| lg | 24px | Card padding (`p-6`), settings section vertical gap |
| xl | 32px | Inter-section gap in Settings page (`space-y-8`) |

## Component Inventory

New components this phase. Reused components listed for executor reference.

### New Components

#### `UserAvatar` (`resources/js/Components/UserAvatar.tsx`)

Props: `name: string`, `avatar?: string | null`, `size?: 'sm' | 'md' | 'lg'`

Size tokens (identical to ClientAvatar):
- sm: `h-8 w-8 text-xs`
- md: `h-10 w-10 text-sm`
- lg: `h-16 w-16 text-xl`

Fallback (no avatar): deterministic 2-color HSL gradient background, white initials on top.

Gradient algorithm (CONTEXT.md D-19 + RESEARCH.md Pattern 5):
```
hash = djb2(name)          // (hash * 31 + charCode) >>> 0
hue1 = hash % 360
hue2 = (hue1 + 130) % 360
from = hsl(hue1, 65%, 52%)
to   = hsl(hue2, 65%, 42%)
background: linear-gradient(135deg, from, to)
```

Avatar photo: `<img src={/storage/${avatar}} className="rounded-full object-cover" />`

Initials: first 2 words' first chars, uppercase, `font-semibold text-white`

Difference from ClientAvatar: gradient fallback (2 colors) vs. flat `#7c3aed/10` tint. Same API contract.

#### `OrgSwitcher` (inline in `AppLayout.tsx`)

Position: header, right side, between user name text and notification bell area. Replaces current `<span>` text user name.

Trigger: `UserAvatar` (size sm) + org name text + `ChevronDown` icon (16px). Max-width 160px, truncate org name.

Dropdown panel: `w-56 rounded-[12px] border border-[#e5e7eb] dark:border-[#1f2937] bg-white dark:bg-[#111827] shadow-lg` — positioned `right-0 top-10 z-50`

Animation: HeadlessUI `Transition` — `ease-out duration-200` enter, `opacity-0 scale-95` → `opacity-100 scale-100` (matches existing Dropdown.tsx pattern)

Dropdown items (each org):
- Layout: `flex items-center gap-3 px-4 py-2.5`
- OrgLogo or org initials (24px circle)
- Org name `text-sm text-[#111827] dark:text-[#f9fafb]`
- Active checkmark: `Check` icon 14px in `text-[#7c3aed]` (right side, only current org)
- Hover: `hover:bg-[#f3f4f6] dark:hover:bg-[#1f2937]`

Divider + "Criar nova organização" as last item — `text-sm text-[#7c3aed] flex items-center gap-2 px-4 py-2.5`

PATCH on item click: `router.patch(route('settings.current-org'), { organization_id: id }, { preserveScroll: false })`

#### `RoleBadge` (inline utility or small component)

Props: `role: 'owner' | 'admin' | 'collaborator'`

Renders per role badge color map above. Pattern identical to StatusBadge.tsx. Copy:
- owner → "Proprietário"
- admin → "Admin"

#### Settings page sub-nav (`resources/js/pages/Settings/Index.tsx`)

Sticky internal nav at top of settings page content area. Appears below the `<h1>` heading.

Layout: `flex gap-1 border-b border-[#e5e7eb] dark:border-[#1f2937] mb-8 sticky top-14 bg-[#f9fafb] dark:bg-[#0b0f14] z-30 -mx-4 md:-mx-6 px-4 md:px-6`

Each tab link: `text-sm font-medium px-1 py-3 border-b-2 -mb-px transition-colors`
- Default: `border-transparent text-[#6b7280] hover:text-[#111827] dark:hover:text-[#f9fafb]`
- Active: `border-[#7c3aed] text-[#7c3aed]`

Tabs: Perfil | Organização | Equipe | IA

Mobile fallback (< 640px): `<select>` dropdown replacing the tab bar. `onChange` scrolls to anchor.

Scroll-spy: IntersectionObserver on each section div, threshold 0.3, updates active tab.

## Page Layout Contracts

### `/settings` (unified settings page)

Route: `GET /settings`
Page component: `resources/js/pages/Settings/Index.tsx` (full replace of stub)

Shell: `AppLayout` with `title="Configurações"`

Content wrapper: `mx-auto max-w-2xl` — matches Settings/Ai.tsx exactly

Section structure (4 anchors):

```
<section id="profile">   <!-- #profile  -->
<section id="organization"> <!-- #organization -->
<section id="team">      <!-- #team     -->
<section id="ai">        <!-- #ai       -->
```

Each section: `rounded-[12px] border border-[#e5e7eb] dark:border-[#1f2937] bg-white dark:bg-[#111827] p-6`
Section heading: `text-sm font-semibold text-[#111827] dark:text-[#f9fafb] mb-4`

Section dividers between panels: `my-8` vertical gap. No visual HR line — gap is sufficient.

### `/settings` — Profile Section (`#profile`)

Fields:
1. **Avatar upload**: circular 64px (`UserAvatar` size lg) + "Alterar foto" button beneath. Click opens file input (hidden). Drag & drop on the avatar circle also accepted.
   - Upload button: `text-sm text-[#7c3aed] hover:text-[#6d28d9] font-medium`
   - Constraint hint: `text-xs text-[#9ca3af]` — "JPG, PNG ou WebP. Máx. 2MB."
2. **Nome**: `TextInput` full-width
3. **Email**: `TextInput` readonly — `bg-[#f9fafb] dark:bg-[#0b0f14] cursor-not-allowed opacity-75`
4. **Alterar senha**: collapsible — collapsed shows "Alterar senha →" link; expanded shows current password + new password + confirm fields
5. **Save button**: `bg-[#7c3aed]` primary — "Salvar perfil"

Layout: avatar + button left-aligned, fields in right column. At mobile: stacked.

### `/settings` — Organization Section (`#organization`)

Fields:
1. **Logo da organização**: 64px org logo circle + "Alterar logo" button (same pattern as avatar upload)
2. **Nome da organização**: `TextInput` full-width
3. **Save button**: "Salvar organização"

Visibility: section visible to all users. Form fields disabled (not readonly) for collaborators with note `text-xs text-[#9ca3af]` "Apenas admins podem alterar as configurações da organização."

### `/settings` — Team Section (`#team`)

Sub-sections stacked vertically with `space-y-6`:

**A. Invite form** (admin/owner only — hidden for collaborators):
- Email input + role select inline row
- Role select: `<select>` with options "Admin" and "Colaborador" (owner cannot be invited via UI)
- "Enviar convite" button
- If collaborator: entire invite card hidden (not shown at all)

**B. Pending invites** (admin/owner only):
- Table/list: email | role badge | "Enviado em {date}" | "Cancelar" + "Reenviar" actions
- Empty: no sub-section shown if no pending invites
- "Cancelar" text: `text-sm text-[#ef4444] hover:text-[#dc2626]`
- "Reenviar" text: `text-sm text-[#7c3aed] hover:text-[#6d28d9]`

**C. Team roster** (visible to all):
- Row: `flex items-center gap-4 py-3 border-b border-[#e5e7eb] dark:border-[#1f2937] last:border-0`
- Left: `UserAvatar` size sm
- Center: name `text-sm font-medium` + email `text-xs text-[#6b7280]`
- Role badge: `RoleBadge` component
- Join date: `text-xs text-[#9ca3af]` — "Desde DD/MM/AAAA"
- Actions (admin/owner only, not on owner row):
  - Role inline change: `Dropdown.Trigger` = current role badge, `Dropdown.Content` = list of role options
  - Remove button: trash icon 16px `text-[#9ca3af] hover:text-[#ef4444] transition-colors` — triggers CostConfirmModal
- Owner row: no actions, owner badge only

### `/settings` — AI Section (`#ai`)

Content: exact copy of current `Settings/Ai.tsx` form — API key card + Managed Agents card + Save button.

Route change: `GET /settings/ai` → 301 redirect to `GET /settings#ai` (CONTEXT.md D-26)

No UI changes to AI section content.

### `/invite/{token}` (Invitation Accept Page)

Route: guest-accessible (unauthenticated)
Component: `resources/js/pages/Invite/Accept.tsx` (new)

Layout: centered card, no AppLayout, no sidebar. Full-page `min-h-screen bg-[#f9fafb] dark:bg-[#0b0f14] flex items-center justify-center p-4`

Card: `max-w-md w-full rounded-[16px] bg-white dark:bg-[#111827] p-8 shadow-lg`

Content:
1. Briefy logo (compact SVG) centered
2. Heading: "Você foi convidado para [Org Name]" — `text-xl font-semibold text-center mb-2`
3. Sub-text: "Você foi convidado como [role badge]" — `text-sm text-[#6b7280] text-center mb-6`

**Branch A — new user (has_account = false):**
- Name input + password input + confirm password input
- Submit: "Criar conta e entrar"

**Branch B — existing user (has_account = true):**
- Info text: "Esta conta já existe no Briefy. Clique abaixo para entrar na organização."
- Submit: "Entrar na organização"

Error state (expired/invalid token): card shows "Este convite expirou ou já foi utilizado." + link "Solicitar novo convite"

Submit button: `w-full bg-[#7c3aed] hover:bg-[#6d28d9] rounded-[8px] py-2.5 text-sm font-medium text-white`

## Interaction Contracts

### Avatar Upload

- Click on avatar circle or "Alterar foto" button → opens hidden `<input type="file">` picker
- Drag & drop also accepted on avatar circle — `onDragOver`/`onDrop` handlers
- On file select: immediate preview via `URL.createObjectURL()` before submit — no round trip
- On submit: `router.post(route('settings.profile.update'), formData, { preserveScroll: true })`
- Loading state: avatar circle shows `Loader2` spinner overlay with `opacity-50` on the image
- Success: `FlashMessage` "Foto atualizada com sucesso."
- Error: `FlashMessage` error variant

### Role Change (Roster)

- Trigger: click current role badge → opens Dropdown with other valid roles
- Owner's own row: no role dropdown (owner is locked)
- Collaborators cannot see role dropdown at all (server-controlled prop + UI hide)
- On select: `router.patch(route('team.updateRole', memberId), { role: newRole }, { preserveScroll: true, only: ['members'] })`
- Loading: disable the badge trigger while patching

### Remove Member

- Click trash icon → opens `CostConfirmModal` (repurposed as confirmation modal)
  - `title`: "Remover membro"
  - `body`: full confirmation copy with member name
  - `confirmLabel`: "Remover"
  - `costUsd`: 0 (renders as "<$0.01" — acceptable repurpose since cost section is secondary)
  - On confirm: `router.delete(route('team.remove', memberId), { preserveScroll: true, only: ['members', 'pending_invites'] })`

### Org Switcher

- Click trigger (UserAvatar + org name) → open dropdown
- Click org item → `router.patch(route('settings.current-org'), { organization_id: id })` → full Inertia reload (page context changes)
- Loading: disable trigger button while PATCH in flight, show spinner on trigger
- Click "Criar nova organização" → navigate to org creation page (deferred — link disabled in Phase 4 with tooltip "Em breve")

### Settings Sub-nav Scroll Spy

- `IntersectionObserver` on section anchors `#profile`, `#organization`, `#team`, `#ai`
- Observer options: `{ threshold: 0.3, rootMargin: '-56px 0px 0px 0px' }` (accounts for 56px header height)
- Click on tab → `element.scrollIntoView({ behavior: 'smooth' })` + update active state immediately (don't wait for observer)
- Mobile: `<select>` `onChange` calls same scroll function

### Invite Form

- Submit: `router.post(route('team.invite'), { email, role }, { preserveScroll: true, only: ['pending_invites'] })`
- On success: clear form fields, `FlashMessage` "Convite enviado para {email}."
- On error (email already member): inline `InputError` on email field
