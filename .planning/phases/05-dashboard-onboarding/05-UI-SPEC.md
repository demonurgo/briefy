## Spacing Scale

Identical to Phase 4 — no deviations. Matches all existing Phase 1–4 components.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding (`px-1`) |
| sm | 8px | Button padding (`px-2 py-2`), card inner gap (`gap-2`) |
| md | 16px | Section inner padding (`p-4`), card padding (`gap-4`) |
| lg | 24px | Card padding (`p-6`), grid gap (`gap-6`) |
| xl | 32px | Inter-section gap (`gap-8`) |
| 2xl | 48px | Dashboard top padding (`pt-6 pb-12`) |
| 3xl | 64px | — (not used this phase) |

Exceptions:
- Status card min-height: 96px (`h-24`) — compact metric display
- Chart container heights: bar/line 200px (`h-[200px]`), donut 160px (`h-[160px]`) within card
- Dashboard grid gap: 24px (`gap-6`) — consistent across all grid regions
- Activity feed max-height: 320px (`max-h-80`) with `overflow-y-auto`
- Date range picker trigger: 36px height (`h-9`) — compact header placement
- Status card delta icon: 14px (`size-3.5`)

**Source:** CONTEXT.md D-04–D-19 + DashboardPlanningWidget.tsx measurements + AppLayout.tsx spacing

## Typography

Inter font throughout. Same four active roles as Phase 4 — no additions.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Table cells, activity feed items, demand list rows, onboarding steps |
| Label | 12px (`text-xs`) | 400 (regular) | 1.4 | Card sub-labels, chart axis labels, badge text, delta values, "Ver todas →" links |
| Heading | 16px (`text-base`) | 600 (semibold) | 1.3 | Card section titles (`"Seu foco agora"`, `"Panorama geral"`, etc.) |
| Display | 24px (`text-2xl`) | 700 (bold) | 1.1 | Status card metric number (demand count) |

Additional roles:
- Greeting name: 18px (`text-lg`) weight 600 — `"Olá, [nome]! 👋"`
- Greeting subtitle: 14px (`text-sm`) weight 400 text-[#6b7280] — contextual subtitle
- Status card label: 12px (`text-xs`) weight 500 (`font-medium`) — card label above the count
- Delta indicator: 12px (`text-xs`) weight 500 — colored up/down text

**Declared weights in this phase: 400 (regular), 500 (medium), 600 (semibold), 700 (bold).**

**Source:** CONTEXT.md D-05, D-07 + 04-UI-SPEC.md typography scale + dashboard reference images

### New Components

#### `DashboardStatusCard` (`resources/js/Components/DashboardStatusCard.tsx`)

Props: `label: string`, `count: number`, `delta: number | null`, `deltaInverted?: boolean`, `icon: LucideIcon`, `iconColor: string`

rounded-[12px] border border-[#e5e7eb] dark:border-[#1f2937]

  - Format: `"↑ 3 vs ontem"` / `"↓ 1 vs ontem"` / `"— mesmo que ontem"`

#### `DashboardSectionCard` (`resources/js/Components/DashboardSectionCard.tsx`)

Generic card wrapper for all dashboard sections.

Layout:
```
rounded-[12px] border border-[#e5e7eb] dark:border-[#1f2937]
bg-white dark:bg-[#111827] p-6
```

Header: `flex items-center justify-between mb-4`
- Title: `text-base font-semibold text-[#111827] dark:text-[#f9fafb]`
- Action (if provided): `text-xs text-[#7c3aed] hover:underline font-medium`

#### `ActivityFeed` (`resources/js/Components/ActivityFeed.tsx`)

Props: `events: ActivityEvent[]`

```ts
interface ActivityEvent {
  id: number;
  action_type: string;
  subject_name: string;
  user_name: string;
  user_avatar?: string | null;
  metadata?: Record<string, unknown>;
  created_at: string; // ISO8601
}
```

Layout: `DashboardSectionCard` wrapper with `max-h-80 overflow-y-auto` on the list

Each event row:
```
flex items-start gap-3 py-3 border-b border-[#e5e7eb] dark:border-[#1f2937] last:border-0
```

Row content:
- Icon circle: `w-8 h-8 rounded-full flex items-center justify-center bg-{color}/10` with action icon (16px, colored)
- Text block: `"[user_name] [verb] [subject_name]"` — text-sm text-[#111827] dark:text-[#f9fafb]
- Timestamp: `text-xs text-[#9ca3af]` — relative (e.g., "há 5 min", "há 2h")

Empty state: `"Nenhuma atividade recente."` — text-sm text-[#9ca3af] text-center py-8

#### `OnboardingChecklist` (`resources/js/Components/OnboardingChecklist.tsx`)

Reuses DashboardPlanningWidget visual pattern exactly.

Props: `hasClients: boolean`, `hasDemands: boolean`, `onDismiss: () => void`

Trigger: show when `!hasClients || !hasDemands` AND `preferences.onboarding_dismissed !== true`

Layout:
```
rounded-xl border border-[#a78bfa]/30 bg-[#7c3aed]/5 dark:bg-[#7c3aed]/10 p-4
```

Header row: `CheckCircle` icon (20px, #7c3aed) + `"Primeiros passos"` title text-sm font-semibold text-[#7c3aed] + dismiss `X` button right-aligned

Steps (2 items):
```
flex items-center gap-3 rounded-lg border border-[#a78bfa]/20 bg-white dark:bg-[#1e1b2e] px-4 py-3
```
- Checkbox: `CheckCircle` (16px, #10b981) if complete, `Circle` (16px, #9ca3af) if pending
- Step text: `text-sm font-medium text-[#111827] dark:text-[#f3f4f6]`
- If pending: `<Link>` in text-xs text-[#7c3aed] to the action URL

Step copy:
1. `"Adicionar um cliente"` → link `/clients/create` (complete when hasClients)
2. `"Criar a primeira demanda"` → link `/demands?create=1` (complete when hasDemands)

Dismiss: `X` button (14px) — `onClick` calls `onDismiss` → PATCH `/settings/preferences` `{ onboarding_dismissed: true }` via `router.patch`

Auto-hide: when `hasClients && hasDemands`, do not render (before PATCH roundtrip)

#### `DashboardViewToggle` (`resources/js/pages/Dashboard.tsx` — inline)

Only rendered for admin/owner (from `auth.user.role`).

Layout: `flex rounded-[8px] overflow-hidden border border-[#e5e7eb] dark:border-[#1f2937]`

Each button: `px-4 py-1.5 text-sm font-medium transition-colors`
- Inactive: `bg-white dark:bg-[#111827] text-[#6b7280] hover:text-[#111827] dark:hover:text-[#f9fafb]`
- Active: `bg-[#7c3aed] text-white`

State: React `useState<'personal' | 'overview'>('personal')` — no URL change, no Inertia reload.

#### `DateRangePicker` (inline in Dashboard.tsx, overview view only — D-13)

Simple two-input range: `<input type="date">` pair.

Layout: `flex items-center gap-2`
- "De:" label text-xs text-[#6b7280]
- Start date input: `h-9 px-3 rounded-[8px] border border-[#e5e7eb] dark:border-[#1f2937] bg-white dark:bg-[#111827] text-sm`
- "–" separator text-[#9ca3af]
- End date input: same classes
- Focus ring: `focus:ring-2 focus:ring-[#7c3aed]/20 focus:border-[#7c3aed]`

Default: current week Monday to Sunday (computed in component on mount).

On change: triggers re-query via Inertia `router.get(route('dashboard'), { start, end }, { preserveScroll: true, preserveState: true, only: ['overview'] })`.

#### Chart Wrappers (inline in Dashboard.tsx or small sub-components)

All use Recharts. Common chart config:
- `ResponsiveContainer width="100%" height={200}` for bar/line
- `ResponsiveContainer width="100%" height={160}` for donut
- Tooltip: `contentStyle={{ background: '#111827', border: '1px solid #1f2937', borderRadius: '8px', fontSize: '12px', color: '#f9fafb' }}`
- Axis labels: `tick={{ fill: '#6b7280', fontSize: 12 }}`
- Grid lines: `stroke="#1f2937"` (dark) / `stroke="#e5e7eb"` (light) — use `CartesianGrid strokeDasharray="3 3"`
- Legend: `text-xs text-[#6b7280]`

**DonutChart** — `PieChart` + `Pie` with `innerRadius={48} outerRadius={72} paddingAngle={3}`
- Center label overlay: absolute positioned `text-lg font-bold` for total count or percentage
- Recharts `Cell` fill from status/priority color map

**BarChart** — `BarChart` + `Bar` with `radius={[4,4,0,0]}` (rounded top)

**LineChart** — `LineChart` + two `Line` components (created: `#3b82f6`, completed: `#10b981`), `dot={false}`, `strokeWidth={2}`

## Interaction Contracts

### View Toggle (D-12)

- Toggle state: `useState<'personal' | 'overview'>` in Dashboard.tsx
- On toggle: switch rendered regions — no Inertia reload, no URL change
- Both views' data pre-loaded as Inertia props on initial page load (personal + overview for admins)
- Collaborators: toggle not rendered; personal view only

### Date Range Picker (D-13)

- Default on mount: `start = Monday of current week`, `end = Sunday of current week`
- On change: `router.get(route('dashboard'), { start, end }, { preserveScroll: true, preserveState: true, only: ['overview'] })`
- Loading: Inertia's built-in progress bar — no local spinner needed

### Demand Table Tabs (D-09)

- State: `useState<'all' | 'in_progress' | 'overdue' | 'completed'>` — filters client-side from prop array
- No Inertia reload — all 4 tab datasets pre-loaded in one prop

### Onboarding Dismiss (D-29)

- Click X: `router.patch(route('settings.preferences'), { onboarding_dismissed: true }, { preserveScroll: true, preserveState: true, only: ['auth'] })`
- Optimistic hide: set local `dismissed` state immediately before PATCH resolves
- After PATCH: `auth.user.preferences.onboarding_dismissed` updated via Inertia shared props

### Chart Tooltips

- Recharts default `<Tooltip>` with custom `contentStyle` (documented in Component Inventory)
- Donut chart: no tooltip on center label — tooltip appears on hover over slice
- Line chart: `activeDot={{ r: 4, fill: color }}` to show active point
