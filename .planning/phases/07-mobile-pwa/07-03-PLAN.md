<task type="checkpoint:human-verify" gate="blocking">
  <name>UAT: Mobile layout at 375px viewport</name>
  <what-built>
    Wave 1 (07-01): AppLayout overflow-x-hidden, KanbanBoard overflow-hidden + responsive height, manifest.json split icon entries + id field.
    Wave 2 (07-02): DemandDetailModal full-screen mobile + slide-up animation + swipe-down gesture + drag handle pill.
    Wave 3 Task 1: app.blade.php and Dashboard.tsx verified — no changes needed.
  </what-built>
  <how-to-verify>
Open the Briefy app in a browser and test the following scenarios. Use Chrome DevTools device emulation set to 375px width (iPhone SE) OR test on a real mobile device.

**MOB-01 — Horizontal scroll fix:**
1. Open the Dashboard page at 375px. Confirm: NO horizontal scrollbar appears along the bottom of the page.
2. Open the Demands/Kanban page at 375px. Confirm: NO horizontal scrollbar on the page itself (the Kanban board may scroll internally, but the page itself should not).
3. Open the Clients page at 375px. Confirm: no horizontal scroll.

**MOB-02 — Mobile usability:**
4. On the Kanban page at 375px: Swipe left/right within the board. Confirm: Kanban columns scroll horizontally (you can reach "in_review", "approved" columns by swiping) WITHOUT the whole page scrolling horizontally.
5. Click any demand card to open DemandDetailModal. Confirm on mobile:
   - Modal opens full-screen (occupies 100% of viewport, no padding on sides)
   - Modal slides up from bottom with a smooth animation (~250ms)
   - A small gray pill/handle is visible at the top of the modal
   - Swiping down more than ~80px from the top of the modal closes it (slides down and dismisses)
   - X button in the header also closes the modal
   - On desktop (resize to 1024px): modal is centered, max-w-6xl, rounded corners, no slide animation — same as before Phase 7

**MOB-03 — PWA installability:**
6. In Chrome at 375px with mobile emulation: Open DevTools → Application → Manifest. Confirm: manifest loads without errors, icons show 4 entries (192 any, 192 maskable, 512 any, 512 maskable), "id" field shows "/dashboard".
7. Optional: In Chrome, look for the "Install app" prompt or the install icon in the address bar. Confirm no "installability" error in the Manifest tab.

**MOB-04 — Service worker stub:**
8. In Chrome DevTools → Application → Service Workers: Confirm the service worker is registered and active. Confirm it is a passthrough stub (no offline caching, requests go through to network normally). This confirms MOB-04 partial coverage — full offline cache is deferred per D-06.
