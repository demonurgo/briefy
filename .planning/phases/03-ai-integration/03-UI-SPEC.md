> Visual and interaction contract for Phase 3 (AI Integration). Generated by gsd-ui-researcher. Consumes 03-CONTEXT.md (D-01..D-28), 03-AI-SPEC.md, docs/design-brief.md. Downstream: planner, executor, ui-checker, ui-auditor.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (established manual token system; shadcn not initialized) |
| Preset | not applicable |
| Component library | none ‚Äî handcrafted components with Tailwind utility classes |
| Icon library | `lucide-react` (18‚Äì20px default) + **`chatbot-icon-{dark,light}.svg`** (mandatory for ALL AI surfaces ‚Äî D-15) |
| Font | Inter (weights 400, 500, 600, 700) loaded via Google Fonts in `resources/css/app.css` |

**Why no shadcn:** Project runs Laravel 13 + Inertia v3 + React 19 with an already-established Tailwind token layer in `tailwind.config.js` (primary palette, status colors, radius scale, shadow scale). Introducing shadcn mid-project would fork the existing pattern language used across Phases 1‚Äì2 (DemandDetailModal, ClientForm, Sidebar, KanbanBoard). All new Phase 3 surfaces must extend the existing token conventions, not replace them.

**Registry Safety Gate:** Not applicable ‚Äî no shadcn or third-party registries in use. All UI is first-party source code.

## Spacing Scale

Declared values (must be multiples of 4). Maps to Tailwind spacing utilities already in use.

| Token | Value | Tailwind | Usage in Phase 3 |
|-------|-------|----------|------------------|
| xs | 4px | `p-1`, `gap-1` | Icon gap inside buttons (`gap-1.5` permitted at 6px for optical alignment with 14px text) |
| sm | 8px | `p-2`, `gap-2`, `rounded-[8px]` | Compact spacing, button/input border radius |
| md | 16px | `p-4`, `gap-4` | Default spacing inside card bodies, form groups |
| lg | 24px | `p-6`, `gap-6` | Modal padding, section padding, tab panel padding |
| xl | 32px | `p-8` | Empty state vertical padding |
| 2xl | 48px | `py-12` | Chat empty state vertical breathing room |
| 3xl | 64px | reserved | Not used in Phase 3 (modal-scoped) |

**Exceptions (documented, inherited from Phase 2):**
- `6px` (`gap-1.5`, `py-1.5`, `px-3.5`) ‚Äî optical alignment for 14px text (established in Phase 2 modal/form).
- `10px` (`py-2.5`, `px-3.5`) ‚Äî input/button medium density (established in Phase 2).
- Chatbot icon sizing ladder: **12px** inline rotating (loading), **16px** inline with label, **20px** as tab icon, **24px** chat message avatar, **32px** dashboard widget, **48px** chat empty state, **64px** brief empty state centerpiece.

## Typography

Four roles, two weights. Matches `text-xs/sm/base/xl` Tailwind scale. Only Inter.

| Role | Size | Weight | Line Height | Tailwind | Usage in Phase 3 |
|------|------|--------|-------------|----------|------------------|
| Micro | 12px | 500 | 1.4 | `text-xs font-medium` | Badge "X posts/m√™s", timestamps in chat, error hints, banner body, redesign feedback placeholder |
| Body | 14px | 400 | 1.5 | `text-sm` | Modal body copy, chat messages, form input text, markdown paragraphs, planning card body |
| Label | 14px | 600 | 1.4 | `text-sm font-semibold` | Tab labels, section headings inside modal, "Gerar Brief" button, planning card titles |
| Heading | 16px | 600 | 1.3 | `text-base font-semibold` | Modal header title, empty state heading, dashboard widget title, planning page `h2` group heading |

**Locked decisions:**
- Only `font-medium (500)` and `font-semibold (600)` weights appear in Phase 3 UI. `font-bold (700)` is reserved for brand/logo and is **not** used in new surfaces.
- Markdown rendering (Brief tab D-06, Chat IA assistant bubbles): `react-markdown` with inherited container classes ‚Äî no `@tailwindcss/typography` plugin. Per-element class mapping:
  - `# H1` ‚Üí `text-base font-semibold mb-2 mt-4 first:mt-0`
  - `## H2` ‚Üí `text-sm font-semibold mb-2 mt-3`
  - `**bold**` ‚Üí `font-semibold`
  - `- / 1.` list ‚Üí `ml-4 list-disc space-y-1 text-sm`
  - paragraph ‚Üí `text-sm leading-[1.5] mb-3 last:mb-0`
  - Inherits text color from container (`text-[#111827] dark:text-[#f9fafb]` for brief; `text-[#6b7280] dark:text-[#9ca3af]` for body support lines)
- Typewriter streaming cursor (D-04, D-11): a 1px-wide block glyph `‚ñé` in primary-500 (`#7c3aed`), `animate-[blink_1s_step-end_infinite]` (keyframes `blink { 50% { opacity: 0 } }` added to `tailwind.config.js`). Cursor is a trailing DOM node, not a string, so it does not pollute saved content. Disappears immediately on stream end.

## Interaction Contract

### Streaming (SSE ‚Äî Shared Contract for Brief + Chat)

| Aspect | Contract |
|--------|----------|
| Transport | `EventSource` ‚Üí `GET /demands/{id}/brief/generate` and `GET /demands/{id}/chat/stream` (per 03-AI-SPEC.md) |
| Chunk granularity | Server may emit 1‚Äì50 chars per event; frontend handles robustly |
| Rendering | Append to React state buffer; one re-render per chunk (60fps fine at ~30 tok/s) |
| Cursor display | `‚ñé` primary-500, step-1s blink, appended as trailing DOM node (not string) so it does NOT pollute the saved content |
| Stream end | Event type `done` or explicit `[DONE]` line; frontend supports both |
| Error during stream | Event type `error` with JSON `{message}` ‚Üí render error banner, preserve partial text, show retry |
| Cancellation | User closes modal / switches tab / clicks a new primary action ‚Üí `EventSource.close()`; server aborts on socket disconnect |
| Persistence | On `done`, server has already persisted. Frontend performs Inertia partial reload `only: ['selectedDemand']` to refresh canonical state. |

### Loading States (no skeletons, no overlays)

| Surface | Pattern |
|---------|---------|
| Brief generate | Header button `Gerando‚Ä¶` + rotating chatbot. Panel clears; typewriter begins. **No skeleton, no overlay** (D-04). |
| Chat send | Input disables; user bubble appears immediately; assistant bubble shows chatbot avatar + blinking cursor ‚Üí fills char-by-char. |
| Planning generate | Modal's submit shows `Gerando planejamento‚Ä¶` + rotating chatbot 16px. No full-page loader. On success: modal closes; new month group slides in at top. |
| Redesign item | Button shows `Aplicando‚Ä¶` + rotating chatbot 12px. Card stays visible with old content. Swap on success with 200ms opacity-fade. |
| Convert single | Button shows `Convertendo‚Ä¶` with lucide `Loader2` spinner 13px. On success: card becomes `accepted` with green check. |
| Convert bulk | Floating bar button shows `Convertendo‚Ä¶`. On success: clears selection + success banner + converted cards flip to `accepted`. |

### Focus & Keyboard

| Surface | Keyboard contract |
|---------|-------------------|
| Chat input | `Enter`=send, `Shift+Enter`=newline, `Esc`=blur (does NOT close modal) |
| Brief edit textarea | `Cmd/Ctrl+S`=save, `Esc`=cancel (confirm if dirty) |
| Planning card bulk-select | `Space`=toggle checkbox when card focused, `Shift+click`=range select |
| Redesign textarea | `Cmd/Ctrl+Enter`=apply, `Esc`=cancel inline edit |
| Widget dismiss | Tabbable, `Enter`=dismiss, `aria-label={t('common.dismiss')}` |
| Generation modal | `Esc`=close, `Enter` inside focused input=submit |

### Accessibility

- All primary CTAs show visible focus rings: `focus:ring-2 focus:ring-[#7c3aed]/40`.
- Chatbot SVG: `alt=""` when paired with text label; `alt={t('ai.assistantIcon')}` when standalone (e.g., tab icon, avatar).
- Tab bar: `role="tablist"`, each tab `role="tab"` + `aria-selected` + `aria-controls`; each panel `role="tabpanel"` + `aria-labelledby`.
- Streaming text container: `aria-live="polite"` so screen readers announce progressively; **no** `aria-live` on the cursor glyph.
- Color contrast: primary-500 on white = 5.6:1 (AA). primary-400 on `#0b0f14` = 7.2:1 (AAA). All text-on-surface combinations pass AA.
- Reduced motion: respect `@media (prefers-reduced-motion: reduce)` ‚Äî disable typewriter cursor blink and rotating chatbot-icon spin; replace with a static state. Typewriter text still streams char-by-char (content-level, not motion).
