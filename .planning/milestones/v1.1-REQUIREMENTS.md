## v1.1 Requirements — Final Status

All 25 v1.1 requirements shipped and verified.

### AI Assistant

- [x] **AI-01**: User can request AI to generate a content brief for a demand
- [x] **AI-02**: AI-generated brief displayed as a dedicated section in the demand detail modal
- [x] **AI-03**: User can regenerate the AI brief with one click
- [x] **AI-04**: User can chat with an AI assistant within a demand
- [x] **AI-05**: AI chat has access to full demand context (metadata, files, comments)
- [x] **AI-06**: AI stores and retrieves client-specific memory via `ai_memory` table
- [x] **AI-07**: AI responses stream progressively (character by character)

### Monthly Planning (AI)

- [x] **MPLAN-01**: Client has a configurable monthly content plan with deliverable quantities
- [x] **MPLAN-02**: Client plan quantities defined when creating a client, editable anytime
- [x] **MPLAN-03**: User can request AI to generate a monthly content planning
- [x] **MPLAN-04**: AI-generated monthly plan produces a structured list of content pieces
- [x] **MPLAN-05**: Generated monthly plan can be viewed, items converted into demands

### Team Management

- [x] **TEAM-01**: Admin can invite a new user by email; invited user joins as a member
- [x] **TEAM-02**: Admin can view all organization members in a team roster
- [x] **TEAM-03**: Admin can change a member's role or remove them
- [x] **TEAM-04**: Any user can edit their own profile: display name, avatar, locale, theme
- [x] **TEAM-05**: Role-based access enforced — only admins/owners can invite, remove, delete content
- [x] **TEAM-06**: Settings page organized into sections: Profile, Organization, Team, AI Key

### Dashboard

- [x] **DASH-01**: Admin/owner sees a dashboard with total demands broken down by status (visual chart)
- [x] **DASH-02**: Admin sees demands per team member (workload distribution)
- [x] **DASH-03**: Admin sees demands per client (client activity overview)
- [x] **DASH-04**: Any user sees their own assigned demands summary (count by status)
- [x] **DASH-05**: Dashboard shows a recent activity feed (last 10–15 demand actions)

### Onboarding

- [x] **ONBRD-01**: First-time user sees a guided checklist: add client → create first demand
- [x] **ONBRD-02**: User can dismiss or mark onboarding as complete at any step
